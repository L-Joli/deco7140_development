Alignment with Client and User Needs

Entry 01 · Scope & Audiences

This implementation operationalises the research findings and user stories from the A2 Design Research findings. It delivers a centralised information hub for the Tech and Innovation Communities, aligning with CCI’s mission to foster social cohesion and equitable access to opportunities. The site brings three core journeys into one place: staying informed (news & events), discovering resources (repositories & guides), and building connections (profiles & messaging).

How the design maps to CCI values

  • Social Cohesion: People can interact and connect through Join Discussions (discuss.html) and Connect with Your Community (connect.html). Staying up to date happens on Home (index.html) and exploration happens on Explore Repositories (explore.html), turning weak ties into strong ties through repeated touchpoints.
  • Inclusivity: Everyone can participate in the discussion forum on Join Discussions (discuss.html). The Learn About Us page (about.html) communicates CCI’s mission and who this community is for, using neutral language and clear participation guidelines.
  • Accessibility: Built to WCAG 2.2 Level AA with: Perceivable (content presented in ways users can perceive), Operable (all UI and navigation usable via keyboard and more), Understandable (plain language and predictable UI), and Robust (compatible with assistive tech). Responsive design adapts from mobile to desktop. More details are in Section 2: Accessibility and Ethical Considerations.

User Needs Supported

  • Centralised Information Hub: From user interviews, participants expressed a strong need for an all-in-one hub combining GitHub-style resources, tech news, and professional connections. The index.html, connect.html, and explore.html pages collectively meet this need by integrating community updates, networking, and resource exploration.
  • Resource Exploration: From user interviews, participants wanted personalised resource discovery. On explore.html, users can browse by topic, see trending repositories, filter by programming language, and search. Relevant items are easy to scan, then open for deeper reading. Key updates also appear on index.html.
  • Professional Networking: From user interviews, participants also wanted a way to connect with others in similar fields. On connect.html, users can discover peers with shared backgrounds/interests and open private conversations.

Addresses the Needs Identified in the A3 Design Proposal

This implementation follows the A3 site map and style guide to maintain both visual and functional consistency across pages. Each page adopts a clear and structured layout with a coherent colour palette to enhance clarity and reinforce the project’s visceral design goals. Interactive states—such as hover, active, and focus effects—provide behavioural feedback that makes the interface intuitive and responsive. Together, these design and structural choices strengthen reflective design by demonstrating how the website’s aesthetics, usability, and purpose align with both user needs and CCI’s values of inclusivity and connection.

  • index.html: Uses genericproduct to populate the home feed and filters items via the event_type field (news, event) for accurate categorisation. Includes loading/empty/error states and an event-detail pattern so users can save items to their profile for later viewing.
  • explore.html: Employs genericproduct API to render resource cards with topic and trending filters, and free-text search. The API was adapted to function as a repository browser, allowing users to explore or submit their own repositories to the community.
  • discuss.html: Integrates multiphotopost API to enable post creation, replies, and moderation/reporting while also retrieving and displaying the discussion list for user interaction.
  • connect.html: Combines genericuserprofile and genericchat APIs to support profile discovery and private messaging.
  • about.html: Presents CCI’s mission, participation norms, and links to privacy & community guidelines.
  • preferences.html: Intended for profile updates and newsletter subscription to support client goals around engagement. As no dedicated API is available, the planned approach is a lightweight email-based subscription workflow (pending client approval of the previous proposal).
  • privacy.html: Communicates transparent data handling, consent, and opt-out options; documents what is stored and why, aligning with ethical standards.

Accessibility and Ethical Considerations

Entry 02 · WCAG & Ethics

Accessibility and ethics were integral to the implementation process, aligning with CCI’s client values of Accessibility and Inclusivity. Each design choice ensures equitable access for all users, reflecting CCI’s commitment to building an inclusive digital community. The website follows WCAG 2.2 Level AA standards to ensure the interface is perceivable, operable, understandable, and robust for diverse audiences. These practices also align with the ACS Code of Ethics principles of Trustworthiness, Respect for Others, and Honesty, demonstrating professional integrity and user respect in digital design.

Accessibility Implementation (WCAG 2.2 AA)

The implementation applies multiple WCAG 2.2 Level AA success criteria through semantic HTML, ARIA attributes, responsive layouts, and interaction design. These ensure that information is accessible and interactions are usable for people with different abilities and assistive technologies.

  • 1.1.1 Non-text Content: All meaningful images have descriptive alt attributes.
  • 1.3.1 Info and Relationships: Landmarks (<header>, <main>, <nav>) and headings convey hierarchy.
  • 1.3.5 Identify Input Purpose: Profile and newsletter fields include autocomplete attributes for clarity and efficiency.
  • 1.4.10 Reflow & 1.4.11 Non-text Contrast: Layouts adapt responsively with strong visual contrast and visible focus states.
  • 2.1.1 Keyboard Accessibility: All interactive elements are reachable and usable via keyboard; focus order follows reading flow.
  • 2.4.1 Bypass Blocks & 2.4.3 Focus Order: Clear landmarks allow quick navigation and maintain logical focus progression.
  • 2.4.4 Link Purpose & 2.4.6 Headings and Labels: Links and buttons use clear, descriptive text and labels.
  • 2.4.7 Focus Visible: Visible focus outlines appear across all pages and states.
  • 2.5.3 Label in Name: The accessible name matches the visible label text for clarity with assistive technologies.
  • 2.5.8 Target Size: All buttons and links meet minimum 24×24px touch target size.
  • 3.2.3 Consistent Navigation: The navigation bar is consistent across all pages.
  • 3.3.1 Error Identification & 3.3.2 Labels or Instructions: Forms include descriptive labels and inline error text.
  • 4.1.2 Name, Role, Value: ARIA roles and states ensure assistive technologies can interpret controls.
  • 4.1.3 Status Messages: Live regions announce dynamic updates politely without disrupting the user’s workflow.
  • 2.3.3 Reduced Motion: Animations respect user preferences through prefers-reduced-motion queries.

These implementations uphold Trustworthiness by ensuring the website functions reliably across devices and assistive technologies, and demonstrate Respect for Others by providing equitable access to all users, regardless of ability.

Ethical Design

Ethical design decisions guided how user data, communication, and inclusivity were implemented. These choices align with CCI’s values of Inclusivity and Social Cohesion, while directly supporting the ACS Code of Ethics principles of Trustworthiness, Respect for Others, and Honesty.

  • Transparency (Honesty): The about.html and privacy.html pages communicate data use in plain language, ensuring users are truthfully informed about how their data and participation are managed.
  • Privacy and Consent (Trustworthiness): Data collection is minimal, with no hidden tracking or analytics beyond necessary functionality. The privacy.html page explicitly explains consent and storage practices, helping users trust the system with informed decisions.
  • Newsletter Subscription (Honesty & Respect for Others): On preferences.html, the newsletter is opt-in only. With no available API, the design uses a simple email-based workflow that respects user privacy and autonomy—upholding transparency and respecting users’ control over their personal information.
  • Representation and Inclusion (Respect for Others): Neutral, inclusive language and imagery avoid stereotypes, fostering a welcoming space that represents diversity across the CCI community.

Audit Summary (IBM Equal Access): Verified compliance on contrast, label associations, and focus indicators. Low contrast issues and redundant roles were corrected, ensuring clean ARIA structure and full keyboard navigation. No critical violations remain.

Evidence and Page Mapping

  • index.html: Dynamic updates announced with ARIA live regions; keyboard-friendly filters; responsive layout with visible focus outlines.
  • explore.html: Structured cards with descriptive headings and filters using real form elements.
  • discuss.html: Accessible post creation form, labeled input areas, and moderation controls.
  • connect.html: Profile cards use semantic headings and accessible buttons for messaging.
  • preferences.html: Forms use fieldsets and legends; newsletter subscription follows ethical opt-in and minimal data collection principles.

Variations and Justifications

Entry 03 · Design refinements

Building on the A3 visual designs and site map, several design refinements were introduced during implementation to improve clarity, usability, and accessibility. Each variation below is justified at the design level with reference to relevant WCAG 2.2 criteria where applicable.

Design & Content

  • Navigation Bar: The logo and navigation links are separated to create a clearer visual hierarchy and more predictable keyboard focus order, aiding orientation for all users. (WCAG 1.3.1 Info and Relationships; 2.4.3 Focus Order)
  • Tooltips & Indicators on nav bar and key elements: Added hover, focus, and active states with brief titles where helpful so interactions provide immediate visual/textual feedback and state clarity. (WCAG 2.4.7 Focus Visible; 1.4.11 Non-text Contrast)
  • Page Titles (verb–noun format): Updated headings to action-oriented titles (e.g., “Join Discussions”, “Connect with Your Community”) to express purpose at a glance and support scanning and wayfinding. (WCAG 2.4.6 Headings and Labels)
  • Cards with badges and richer metadata: Card layouts now surface key attributes (e.g., type/status badges) to improve information scent and reduce cognitive load while preserving clean hierarchy and reflow. (WCAG 1.3.1 Info and Relationships; 1.4.10 Reflow)

Outcome

These variations strengthen visual clarity, reinforce consistent patterns, and make interactions easier to perceive and operate, especially for keyboard and assistive-technology users. Overall, the refinements keep fidelity with the A3 direction while elevating usability and alignment with WCAG 2.2 Level AA.

Challenges and Strategies

Entry 04 · Implementation Reflection

During the implementation phase, several technical and design challenges emerged that required iterative problem-solving. This section reflects on those key challenges and the strategies employed to address them, demonstrating adaptive thinking and adherence to accessibility and usability principles.

Challenge 1: API Limitations and Dynamic Data

Some APIs provided limited or placeholder data structures that constrained intended dynamic features. For example, the genericproduct and multiphotopost endpoints did not return full metadata or required authentication for certain operations.

Strategy: Implemented mock and filtered data models using available fields (e.g., event_type for news/events). This ensured consistent user experience while maintaining scalability for future integration. Loading indicators and error messages were added to communicate network or data issues, improving transparency and user trust.

Challenge 2: GenAI Prompting and Instruction Clarity

Working with generative AI tools introduced unique challenges, particularly in achieving the desired responses through a single prompt. Early iterations often resulted in incomplete or misaligned outputs due to insufficient contextual details provided at once.

Strategy: Learned to provide comprehensive, structured instructions upfront—clearly outlining objectives, technical context, and design constraints. This approach reduced ambiguity, improved response accuracy, and streamlined collaboration efficiency with AI-assisted workflows.

Challenge 3: Responsive Design and Content Reflow

Creating a responsive interface for various screen sizes without compromising readability and interaction required significant layout testing.

Strategy: Adopted a mobile-first approach with relative units and flexible grid layouts. Ensured reflow integrity for text and components under zoom and small viewport conditions, satisfying WCAG 1.4.10 Reflow and improving accessibility on mobile devices.

Reflection

Overcoming these challenges reinforced the importance of iterative testing, user-centered design, and adaptive communication methods. Each obstacle informed stronger design and collaboration practices—ensuring that the final website was inclusive, responsive, and aligned with client and user needs. This process demonstrated professional problem-solving within realistic implementation constraints.
